# Highlight Reviews

A Contentful sample app that lets stakeholders highlight text on a preview site and leave feedback — as Tasks or Comments on the corresponding Contentful entry — without needing a Contentful login.

> **Disclaimer:** Contentful provides this sample code solely to demonstrate a technical scenario. Any and all sample code provided by Contentful is not intended for production use. Contentful is not responsible for maintaining or supporting this sample code after it has been provided to you.

---

## How it works

1. A stakeholder visits your preview URL.
2. They select any text on the page — a popover appears asking for their feedback.
3. They choose to create a Task or add a Comment, and optionally assign it to a team member.
4. The feedback appears instantly on the relevant entry in Contentful, linked to the specific field where the highlighted text lives.

No Contentful account required for the reviewer. All CMA calls are proxied through a Vercel serverless function that holds the credentials.

---

## Architecture

```
overlay/          Vanilla JS bundle injected into any preview site
src/              Contentful app (ConfigScreen + Sidebar) — React/Vite
server/
  api/
    annotations.ts   POST /api/annotations — creates Tasks or Comments via CMA
    users.ts         GET  /api/users       — returns space members for assignee picker
  public/
    index.html       Patched demo site (generated by build:demo)
    overlay.js       Built overlay bundle
demo-site/        Demo site source (uses __PLACEHOLDER__ tokens)
scripts/
  setup.ts         Interactive setup — creates content types and seeds demo content
  build-demo.ts    Patches placeholders from .env.demo into server/public/index.html
```

## Setup

### Prerequisites

- A Contentful space with a CMA token (Personal Access Token)
- A Contentful Delivery API token for the same space
- A [Vercel](https://vercel.com) account for hosting the proxy and demo site

### 1. Deploy the server to Vercel

```bash
cd server
vercel --prod
```

Set the following environment variables on the Vercel project:

| Variable         | Description                              |
|------------------|------------------------------------------|
| `CMA_TOKEN`      | Contentful Personal Access Token         |
| `SPACE_ID`       | Your Contentful space ID                 |
| `ENVIRONMENT_ID` | Environment ID (default: `master`)       |
| `ALLOWED_ORIGIN` | Your preview site origin (or `*`)        |

### 2. Run the setup script

```bash
npm run setup
```

This will prompt for your space credentials and API base URL, create the demo content types and seed entries, patch the demo site, and trigger a Vercel redeploy.

### 3. Add the overlay to your preview site

Copy the snippet from the deployed demo site or add it manually:

```html
<script>
  window.HighlightReviewsConfig = {
    apiBase: 'https://your-vercel-deployment.vercel.app',
    spaceId: 'your-space-id',
    environmentId: 'master',
    enableTasks: true,
    enableComments: true,
    locale: 'en-US',           // optional, defaults to en-US
    reviewerName: 'Stakeholder', // optional
  };
</script>
<script src="https://your-vercel-deployment.vercel.app/overlay.js"></script>
```

For best results, add `data-contentful-entry-id` and `data-contentful-field-id` attributes to your preview site's HTML elements (the [Contentful Live Preview SDK](https://www.contentful.com/developers/docs/tools/live-preview/) does this automatically).

## Development

```bash
npm install
npm run build:overlay   # build overlay.js
npm run build:demo      # patch demo site from .env.demo
```

The Contentful app (ConfigScreen/Sidebar) can be run locally:

```bash
npm start
```

## npm scripts

| Script             | Description                                              |
|--------------------|----------------------------------------------------------|
